# 市场能力层架构

本文档定义 AlphaFlow 的公共市场能力层边界。目标是让趋势、波动、结构、位置、事件和质量能力只开发一次，由多个策略、在线引擎和回测引擎共同消费，避免在每个策略中重复实现市场分析。

本文同时包含当前实现和目标架构。标注为“目标”的接口尚未全部实现，不能视为当前运行协议。

## 核心原则

- 能力层描述市场事实和状态，不产生开仓、平仓或仓位结论。
- 策略负责触发、能力组合、阈值、仓位、退出和策略诊断。
- 在线与回测使用同一套指标和能力计算，禁止维护回测专用近似公式。
- 能力计算只能读取观察时点及之前的数据，未来收益只用于离线验证。
- 一个能力可以被多个策略以不同方式消费，但同名字段的计算语义必须一致。
- 基础连续值、离散状态、状态年龄和可信度分开输出，不用单一总分替代原始信息。
- 综合评分是能力层的消费者，不反向决定基础能力如何计算。

## 分层模型

```text
K 线事实数据
  -> 基础指标计算
  -> 通用指标窗口
  -> 单周期市场能力
       -> 趋势能力
       -> 波动能力
       -> 结构能力
       -> 位置能力
       -> 事件能力
       -> 质量能力
  -> 多周期市场上下文
  -> 策略消费
       -> Supertrend
       -> Breakout
       -> Pullback
       -> Mean Reversion
       -> 后续模型策略
```

当前实现中，基础指标位于 `backend/go-service/pkg/indicatorcalc`，窗口统计和大部分语义特征位于 `backend/go-service/pkg/indicatorwindow`，在线与回测上下文由 `backend/go-service/pkg/strategyframe` 组装。第一阶段继续沿用这些边界，不立即新建大型框架。

## 六个能力域

| 能力域 | 负责描述 | 不负责 |
| --- | --- | --- |
| 趋势 | 方向、强度、ATR 标准化斜率、效率、持续时间、多周期一致性 | 是否允许某策略追涨或做空 |
| 波动 | ATR 百分比、ATR 分位数、扩张/收缩、异常波动、波动阶段 | 仓位大小或是否止损 |
| 结构 | HH/HL、LH/LL、BOS、CHoCH、区间边界和结构年龄 | 某策略是否将 BOS 作为入场触发 |
| 位置 | 距均线、VWAP、摆动点、关键位的 ATR 距离和区间位置 | 当前位置是否满足某策略性价比要求 |
| 事件 | 翻转、突破、回踩、reclaim、假突破、交叉及事件年龄 | 事件是否直接开仓 |
| 质量 | 数据覆盖率、新鲜度、连续性、冲突度、独立来源数和可信度 | 用一个黑盒分数替代所有能力 |

## 能力字段契约

一个基础能力优先输出以下维度：

```text
value       连续原始值或标准化数值
state       可解释的离散状态
age         当前状态持续时间或事件发生年龄
confidence  数据覆盖和独立证据可信度
```

例如趋势方向可以输出：

```text
trend_capability_direction_score
trend_capability_direction_state
trend_capability_direction_age_bars
trend_capability_direction_confidence
trend_capability_direction_conflict
trend_capability_direction_sources
```

字段约定：

- 连续值必须保留，离散状态不能成为唯一输出。
- `confidence` 描述测量可信度，不等同于策略胜率。
- `age` 使用已闭合 K 线数量或明确的时间单位。
- 缺失数据不得静默转换成中性或零强度；应通过覆盖率或有效来源数显式表达。
- 多个高度相关来源不能简单重复计票，应记录来源分组和冲突度。

## 单周期与多周期边界

单周期能力只读取一个周期的历史窗口，适合在 `pkg/indicatorwindow` 中生成。例如当前周期趋势方向和持续时间、ATR 标准化斜率、路径效率、波动阶段、结构状态和事件年龄。

多周期能力读取多个已闭合周期的单周期能力，由共享上下文组装层生成。例如 3m、5m、10m、15m、30m 趋势一致性，对齐、反向和中性周期数量，多周期覆盖率及结构冲突。

多周期公共能力不得输出 `blocking`、`entry_allowed` 或 `should_buy`。这些名称包含策略语义。公共层只输出客观的一致、反向、中性和可信度统计。

## 策略职责

策略只维护：

- 策略特有触发，例如 Supertrend flip 或 wick reclaim；
- 选择消费哪些公共能力；
- 策略阈值和能力组合方式；
- 仓位计划和风险预算；
- 止盈、止损、移动保护和退出；
- 策略特有的诊断原因。

策略不得重复计算通用趋势、波动、结构、位置、事件质量和多周期一致性。同一能力可以被不同策略按不同方式使用：趋势策略可以要求较高方向一致性，回调策略可以在趋势有效时寻找位置回撤，均值回归策略可以选择低效率和区间边缘；这些差异属于策略规则，不需要复制能力计算。

## 版本与兼容性

能力板块独立版本化，避免一个整体版本掩盖不同板块的语义变化：

```text
trend-capability.v1
volatility-capability.v1
structure-capability.v1
location-capability.v1
event-capability.v1
quality-capability.v1
```

- 新增可选字段可以保持版本兼容。
- 改变现有字段公式、范围或状态含义必须升级对应板块版本。
- 回测观察标签和交易诊断应记录实际能力版本。
- 策略升级能力版本前必须重新回放验证，不能静默继承新公式。
- 第一阶段能力计算参数固定在公共实现中；验证稳定后再决定是否提供公共能力配置。
- 策略配置只定义如何消费能力，不重新定义同名能力的计算公式。

## 独立验证

能力验证与策略回测分离。固定时间采样不依赖策略触发、是否开仓或交易结果，标签只用于离线评价。

每个能力板块按以下顺序推进：

1. 明确测量对象、公式、范围和缺失语义。
2. 实现多个相互独立的基础特征，不先合成总分。
3. 使用固定时间观察样本生成未来路径标签。
4. 检查分桶梯度、方向对称、非重叠样本和时间外窗口。
5. 做消融和相关性分析，证明新增特征具有增量信息。
6. 记录适用行情、失效行情和数据覆盖要求。
7. 通过后冻结定义，再由策略做独立消费实验。

可信等级：

| 等级 | 含义 |
| --- | --- |
| Experimental | 定义和测试完成，仅有开发区间结果 |
| Candidate | 至少一个未参与定义的时间外窗口有效 |
| Stable | 多个滚动窗口和行情阶段表现稳定 |
| Strategy-ready | 成本后有效，边界和失效条件明确，可进入策略实验 |
| Deprecated | 无增量信息、语义不稳定或已被替代 |

综合评分只能优先组合 `Stable` 以上能力；策略硬过滤应优先使用 `Strategy-ready` 能力。

## 当前状态与迁移路线

当前已经实现：

- 在线与回测共享 `indicatorcalc`、`indicatorwindow` 和 `strategyframe`。
- 窗口层已经输出趋势、均线、动量、波动、结构和事件类语义字段。
- 回测支持策略无关的固定时间市场观察标签。
- 市场综合能力 v2.7 已能独立输出方向、强度、风险、延伸风险和数据覆盖信息。

当前不足：

- 部分公共市场判断仍位于 Supertrend 策略文件中。
- 能力板块尚未全部使用独立版本。
- `state/value/age/confidence` 尚未在所有能力域统一。
- 多周期方向分类仍混有 `blocking` 等策略语义。
- 当前综合强度尚未证明可作为稳定策略准入条件。

迁移采用“验证一个、迁移一个”，不一次性重写 Supertrend：

1. 建立并验证公共趋势能力。
2. 让 Supertrend 旁路读取新能力，对比旧判断但不立即改变决策。
3. 新能力达到 `Strategy-ready` 后，迁移并删除策略内对应重复计算。
4. 按波动、结构、位置、事件、质量依次重复。
5. 新策略只能消费已稳定公共能力或明确标记的实验能力，不复制旧策略实现。
6. 当多个策略稳定复用同一契约后，再评估是否从 `pkg/indicatorwindow` 提取独立 `pkg/marketcapability`。
