# 通用市场状态识别

`pkg/marketregime` 提供与具体策略无关的多周期市场状态检测。检测器不读取 Supertrend 或其他策略信号，只消费已收盘K线以及通用的 ATR、成交量、均线和 MACD 指标。

## 版本化公共接口

模块通过 `Analyzer` 暴露统一的流式分析接口；每个版本按时间顺序消费已收盘K线并输出相同的 `Result`。`NewAnalyzer` 用版本号选择实现，现有 `NewDetector` 保留为 v1 兼容入口。当前仅注册 `v1`，后续版本可以并行回放，不替换已有模型。

输出同时保留兼容字段 `AllowNewPosition`，并增加方向独立的 `AllowLong`、`AllowShort`。在 v1 中，`normal` 允许双向；`trend_armed` 和 `trend_active` 只允许已确认趋势方向；其他状态双向暂停。这些字段仍然只是通用市场许可标签，不是策略入场信号，也未接入线上执行。

## 状态与权限语义

| 状态 | 含义 | 新仓建议 |
| --- | --- | --- |
| `normal` | 没有形成全局否决条件 | 保留原策略权限 |
| `chop_watch` | 横盘证据正在累积 | 暂停新仓 |
| `chop_lock` | 高周期萎靡或本周期长期停滞 | 暂停新仓 |
| `breakout_pending` | 首次放量突破，等待站稳和高周期动能 | 暂停追单 |
| `trend_armed` | 突破站稳且高周期动能同向 | 允许对应方向 |
| `trend_active` | 已确认趋势仍在延续 | 允许对应方向 |
| `failed_breakout` | 突破重新收回平台 | 暂停并冷却 |

`AllowNewPosition` 只是研究输出，不代表已经获得在线交易授权。当前没有接入策略引擎或执行引擎。

## v1 证据与状态机

默认使用15m和30m两个高周期。每个周期对均线缠绕、EMA25/EMA99间距、MACD离零轴距离、MACD柱体大小和EMA25斜率投票；价格平台另外使用最近60根3m K线的效率比和平台宽度。所有距离均用15m ATR归一化。

横盘锁定有两种来源：

- `higher_timeframe_dormant`：两个高周期分别达到至少三项萎靡证据，同时价格低效率且平台宽度有限。
- `local_stall`：即使高周期仍保留旧趋势，只要最近窗口效率比不高于0.10且宽度不高于3.5个15m ATR，也按长期停滞处理。

横盘证据连续10根3m K线后锁定。首次突破要求 `volume_ratio20 >= 1.5`，之后至少连续两根收在平台外；同时至少一个高周期必须满足均线方向、MACD柱体方向一致、MACD轴距超过0.3 ATR且柱体超过0.08 ATR，才进入 `trend_armed`。假突破回到平台后进入5根冷却。

## 2026-05-01 至 2026-06-08 冻结参数验证

保持 v5 全部参数不变，在包含上涨、反弹和瀑布的独立窗口回放。18,660根观察中54.84%被标记为禁止新仓。研究候选联表结果如下：

| 止损 / 止盈 | 禁入组净结果 | 放行组净结果 |
| --- | ---: | ---: |
| 50% / 75% | -14.31 bps | -16.61 bps |
| 150% / 150% | -5.33 bps | -16.79 bps |

净结果均已扣除16 bps往返成本。独立窗口中禁入组反而优于放行组，说明“横盘/等待状态”不能被解释成对所有策略都负期望；平台边缘反转、区间交易和假突破反向机会会被全局硬过滤一起删除。

## 当前结论

v1 可以作为通用市场环境标签和研究能力，但全局交易门控验证失败，不得直接接入线上策略。后续策略应分别声明自己如何消费状态，例如趋势策略在 `chop_lock` 中停用，而经过独立验证的区间策略可能只在该状态运行。任何权限规则都必须使用单持仓、手续费、滑点和退出冷却进行独立验证，不能仅凭横盘覆盖率上线。

## v2 与 v3 版本对照

v2 将趋势可交易性和方向拆开。趋势可交易性由价格效率、ADX、ATR归一化均线色带宽度及扩张状态组成；方向由EMA7/25/99排列、KAMA斜率和量价资金方向组成。有效多头色带扩张时只保留多头许可，空头完全对称。所有方向变化连续确认后生效。

v3 在v2上增加独立实现的ATR动态噪声过滤器。首次实现将过滤线静止视为全局禁止，导致2026-07窗口89%以上K线被锁定；参数扫描中提高确认时间进一步将锁定比例推高到96%—99%，证明单项噪声证据不能作为硬否决。修正版仅将静止作为一票证据，在ATR方向与v2方向冲突时进入等待，同向时保留顺势许可。

### 单持仓研究口径

研究命令额外运行一个不接入线上策略的单持仓回放：同一时间最多一个ETH仓位，持仓期间忽略重复候选，同根K线出现多空候选时跳过。入场必须获得所选市场状态版本的方向许可。默认使用100 USDT保证金、100倍杠杆、50 bps初始止损；达到50 bps盈利后把保护位提高到覆盖16 bps往返成本，达到75 bps后按最高有利波动回撤30 bps移动保护。保护位只在下一根K线生效，同根K线存在顺序歧义时不使用事后最高价优化。最长持仓12小时，退出后冷却2根3分钟K线。

单持仓回放记录每笔交易的最大有利波动（MFE）、最大不利波动（MAE）、退出原因和入场市场分数。2026-05-01至06-08的92笔亏损分布为：

| 类型 | 数量 | 亏损单占比 | 含义 |
| --- | ---: | ---: | --- |
| 几乎无利润 | 39 | 42.39% | MFE低于16 bps，入场后没有覆盖成本的空间 |
| 小利润后止损 | 51 | 55.43% | MFE介于16与50 bps之间，未达到保本启动线 |
