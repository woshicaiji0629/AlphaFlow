# 通用市场状态识别

`pkg/marketregime` 提供与具体策略无关的多周期市场状态检测。检测器不读取 Supertrend 或其他策略信号，只消费已收盘K线以及通用的 ATR、成交量、均线和 MACD 指标。

## 版本化公共接口

模块通过 `Analyzer` 暴露统一的流式分析接口；每个版本按时间顺序消费已收盘K线并输出相同的 `Result`。`NewAnalyzer` 用版本号选择实现，现有 `NewDetector` 保留为 v1 兼容入口。当前仅注册 `v1`，后续版本可以并行回放，不替换已有模型。

输出同时保留兼容字段 `AllowNewPosition`，并增加方向独立的 `AllowLong`、`AllowShort`。在 v1 中，`normal` 允许双向；`trend_armed` 和 `trend_active` 只允许已确认趋势方向；其他状态双向暂停。这些字段仍然只是通用市场许可标签，不是策略入场信号，也未接入线上执行。

## 状态与权限语义

| 状态 | 含义 | 新仓建议 |
| --- | --- | --- |
| `normal` | 没有形成全局否决条件 | 保留原策略权限 |
| `chop_watch` | 横盘证据正在累积 | 暂停新仓 |
| `chop_lock` | 高周期萎靡或本周期长期停滞 | 暂停新仓 |
| `breakout_pending` | 首次放量突破，等待站稳和高周期动能 | 暂停追单 |
| `trend_armed` | 突破站稳且高周期动能同向 | 允许对应方向 |
| `trend_active` | 已确认趋势仍在延续 | 允许对应方向 |
| `failed_breakout` | 突破重新收回平台 | 暂停并冷却 |

`AllowNewPosition` 只是研究输出，不代表已经获得在线交易授权。当前没有接入策略引擎或执行引擎。

## v1 证据与状态机

默认使用15m和30m两个高周期。每个周期对均线缠绕、EMA25/EMA99间距、MACD离零轴距离、MACD柱体大小和EMA25斜率投票；价格平台另外使用最近60根3m K线的效率比和平台宽度。所有距离均用15m ATR归一化。

横盘锁定有两种来源：

- `higher_timeframe_dormant`：两个高周期分别达到至少三项萎靡证据，同时价格低效率且平台宽度有限。
- `local_stall`：即使高周期仍保留旧趋势，只要最近窗口效率比不高于0.10且宽度不高于3.5个15m ATR，也按长期停滞处理。

横盘证据连续10根3m K线后锁定。首次突破要求 `volume_ratio20 >= 1.5`，之后至少连续两根收在平台外；同时至少一个高周期必须满足均线方向、MACD柱体方向一致、MACD轴距超过0.3 ATR且柱体超过0.08 ATR，才进入 `trend_armed`。假突破回到平台后进入5根冷却。

## 当前结论

v1 可以作为通用市场环境标签和研究能力，但全局交易门控验证失败，不得直接接入线上策略。后续策略应分别声明自己如何消费状态，例如趋势策略在 `chop_lock` 中停用，而经过独立验证的区间策略可能只在该状态运行。任何权限规则都必须使用单持仓、手续费、滑点和退出冷却进行独立验证，不能仅凭横盘覆盖率上线。

## v2 与 v3 版本对照

v2 将趋势可交易性和方向拆开。趋势可交易性由价格效率、ADX、ATR归一化均线色带宽度及扩张状态组成；方向由EMA7/25/99排列、KAMA斜率和量价资金方向组成。有效多头色带扩张时只保留多头许可，空头完全对称。所有方向变化连续确认后生效。

v3 在v2上增加独立实现的ATR动态噪声过滤器。首次实现将过滤线静止视为全局禁止，导致2026-07窗口89%以上K线被锁定；参数扫描中提高确认时间进一步将锁定比例推高到96%—99%，证明单项噪声证据不能作为硬否决。修正版仅将静止作为一票证据，在ATR方向与v2方向冲突时进入等待，同向时保留顺势许可。

### v4进场过滤与趋势回踩恢复验证

### v5压缩突破快速解锁实验

2026-07-19 针对v4在7月14日主升段长期停留于`chop_lock`的问题，新增独立v5研究版本，未覆盖v4。v5只在锁定状态检查最近20根3分钟K线区间突破；价格突破、布林带扩张、v2方向分数、Squeeze Momentum方向和增量、`volume_ratio20 >= 1.2`全部同向时立即进入`trend_armed`，不等待普通两根状态确认。任一条件不足继续保持锁定，并分别记录带宽、方向、动量和成交量不足原因。

ETHUSDT 2026-07-01 00:00至07-18 13:30 UTC使用与v4相同的单持仓和成本参数。标准Supertrend纯翻转结果从v4的10笔、5胜5负、+38.19 USDT、PF 1.116、最大回撤1.28%，变为v5的21笔、11胜10负、+168.74 USDT、PF 1.256、最大回撤1.99%。该窗口收益有所改善，但没有完成实验的主要目标：7月14日03:47 UTC、价格1782.01的多头翻转仍被原压缩逻辑锁定；08:29 UTC、价格1785.59的多头翻转虽然突破近期区间，却因当根`bb_width_state`不是`expanding`被`v5_breakout_width_weak`拦截。两处信号截至当日结束的最大上涨分别为6.44%和6.23%，最大先行回撤仅0.25%和0.21%。v5直到18:38 UTC、价格1874.24才放行多头，主升段已经运行约5%。

因此v5快速解锁的当前定义失败：单一K线的离散`bb_width_state`与价格突破没有稳定同时出现，造成确认条件交集过窄。虽然7月聚合收益改善，但它来自其他新增交易，不能证明目标行情覆盖改善；按预设停止2025和2026长窗口验证，不继续调整阈值，也不接入正式策略。下一步若继续，应先研究“带宽连续扩张/宽度斜率”替代“当根必须标记expanding”，并对假突破样本做成对消融。

### v6连续带宽与延迟激活实验

2026-07-19 在v5之外增加独立v6。v6将快速解锁的带宽条件扩展为：离散状态为`expanding`、`bb_width_pct`连续至少两根上升或`bb_width_delta > 0`三者满足其一。研究比较器同时为标准、自适应和AI Supertrend增加“延迟激活”回放：因压缩被拦截的翻转最多保留10根3分钟K线；v6同向许可后，只有价格仍沿信号方向且距离原信号不超过1 ATR时才进场；反向翻转、非压缩拦截或超时会取消待确认信号。

同一7月窗口中，标准Supertrend使用v6直接翻转为22笔、12胜10负、+211.39 USDT、PF 1.320、最大回撤1.98%；延迟激活模式为28笔、14胜14负、-22.76 USDT、PF 0.975、最大回撤2.95%。延迟模式增加了交易和亏损，未形成正期望。逐笔确认显示，7月14日08:29 UTC、1785.59的多头信号在后续10根K线内仍未获得v6许可并最终过期；延迟模式当日仍只在18:38 UTC、1874.24入场。因此连续带宽证据没有把目标主升段提前覆盖。

v6实验结论为失败：直接翻转模式的7月收益继续改善，但目标行情仍未覆盖；延迟激活模式PF低于1，证明简单保留被拦截信号会重新引入低质量交易。按预设不运行2025和2026长窗口，不扩大等待时间，不接入正式策略。后续若继续研究，应先把“价格已有效离开压缩区”本身定义为独立入场事件，而不是继续依赖Supertrend翻转与市场状态许可在有限窗口内重新重合。

### 压缩区独立突破事件实验

2026-07-19 新增研究专用`compression_range_breakout`事件，复用3m/5m标准Supertrend同向检查和单持仓保护。检测器进入`chop_lock`时冻结此前20根3分钟K线的高低区间，锁定期间收盘突破冻结边界后，还要求v2方向分数绝对值至少35、Squeeze Momentum及增量同向、`volume_ratio20 >= 1.2`；每个锁定周期最多消费一次事件，同方向冷却20根。该事件使用研究许可绕过产生它的横盘锁定，未修改线上策略或正式市场状态。

同一7月窗口共出现21个冻结区间突破事件，单独回放为21笔、8胜13负、-580.12 USDT、PF 0.324、最大回撤5.85%；标准翻转与压缩突破合并为41笔、19胜22负、-373.52 USDT、PF 0.743、最大回撤5.10%。对照的v6标准Supertrend纯翻转为22笔、12胜10负、+211.39 USDT、PF 1.320、最大回撤1.98%；原有`trend_platform_breakout`在相同单持仓口径下为111笔、51胜60负、-1942.45 USDT、PF 0.503。冻结边界提高了覆盖，但增加的13笔亏损和更高回撤说明假突破过多。

逐事件时间检查显示，7月14日08:29 UTC的目标多头启动仍未当时捕获；同日相关多头事件到10:11:59 UTC才出现，晚约102分钟。7月16日有多头事件，但不能弥补7月14日的覆盖失败和整体负期望。这说明主要瓶颈已不是区间边界滚动，而是“必须已处于`chop_lock`”与方向、动量、量能、双周期Supertrend同时成立的交集仍然偏晚；一旦放行更多冻结区间突破，假突破又明显增加。该实验同时在关键行情覆盖和收益两项验收中失败，因此按预设门槛不运行2025/2026长窗口，不接入正式策略。

### 2025年8月延续入场与AI灵敏度研究收尾

为解决单趋势退出后无法再次参与和高波动月份订单偏少的问题，研究依次测试了同趋势3m结构重入、独立3m BOS、次根确认、5m BOS后3m回踩收复，以及同一5m趋势阶段只补一次。宽松模式把成交放大到约180至360笔，PF集中在`0.36-0.60`；收紧到“必须先退出原仓位”后，标准和自适应分别从38笔增加到53、54笔，但净收益恶化到`-701.05`和`-651.16`。AI只增加2笔，净收益由`+19.68`升至`+33.21`，样本和增量都不足。上述延续入口均已从研究代码删除。

对2025年8月前20个12小时大波段的复核显示，AI实际交易只覆盖5个；15个漏单中，8个在波段内没有同向AI flip，7个虽出现同向flip但被市场状态拦截，没有发现已经放行却因持仓或冷却漏单的大波段。拦截原因主要是`v4_compression_locked`和`v6_state_pending`各4次，另有1次`v6_breakout_direction_weak`。部分flip接近波段末端，说明少单的首要原因是AI指标延迟，而不只是市场状态过严。

将AI因子候选上限从`1-5`缩到`1-4`后，原始flip从119增至170，20个波段的同向flip覆盖从12个增至15个，首次同向flip中位延迟从304分钟降至217分钟；但实际回放只有16笔、7胜9负、`-439.90 USDT`、PF `0.259`。`1-3.5`产生219个flip，13笔实际交易为`-252.48 USDT`、PF `0.454`。快速版本增加的是震荡翻转，不是高质量入口，已经删除，原AI `1-5`保持不变。

研究结论不是继续放宽市场状态或制造更多独立事件，而是先对全部AI flip（包括被拦截信号）使用统一前向标签，寻找高周期方向、结构、量能和动量是否存在跨窗口稳定的正期望子集。未完成该验证前，不调整线上市场状态，不把订单数量作为独立优化目标。
